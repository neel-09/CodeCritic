This document defines the Domain Specific Language (DSL) used by the Planner node to generate test specifications. These assertions will be evaluated against the VCD (Value Change Dump) file generated by the Wokwi hardware emulation.

## Assertions

The following assertions are supported:

### 1. TimingAssertion
Evaluates the frequency and regularity of a signal on a specific digital pin.

* **What it checks:** Does a pin toggle at a regular, predictable interval?
* **Inputs needed:**
  * `pin_number` (integer/string): The target hardware pin (e.g., 13).
  * `expected_period_ms` (float): The expected time between state toggles in milliseconds.
  * `tolerance_ms` (float): The acceptable margin of error to account for simulation quirks or microcontroller clock cycles.
  * `min_toggles` (int): minimum number of toggles expected in the simulation window. If the VCD has fewer toggles than this, fail immediately without checking periods.
* **Pass condition:** Every state toggle captured in the VCD data for the target pin falls strictly within `expected_ms ± tolerance_ms`.


### 2. LevelAssertion
Evaluates the absolute voltage state of a pin at a precise moment in time.

* **What it checks:** Is a pin HIGH or LOW at a specific time?
* **Inputs needed:**
  * `pin_number` (integer/string): The target hardware pin.
  * `expected_level` (string): The required electrical state (`HIGH` or `LOW`).
  * `timestamp_ms` (float): The exact simulation time in milliseconds to evaluate the pin state.
* **Pass condition:** The parsed VCD data confirms that the pin's value at `timestamp_ms` exactly matches the `expected_level`. pin state is the last recorded value at or before timestamp_ms.


### 3. ProtocolAssertion
Evaluates complex digital communication waveforms to verify peripheral communication.

* **What it checks:** Did an I2C, SPI, or UART device respond correctly to a command?
* **Inputs needed:**
  * `protocol_type` (string): The communication bus being tested (e.g., `I2C`, `SPI`, `UART`).
  * `address` (hex/string): The peripheral device address on the bus.
  * `expected_response` (string/byte array): The data payload expected from the peripheral.
* **Pass condition:** The protocol-specific parser successfully decodes the VCD waveform and matches the decoded payload against the `expected_response`.
* **Note**: Mark it as "not implemented in v1 — placeholder for future". Only implement TimingAssertion and LevelAssertion in evaluator.py


* **Rules** -
  * If VCD data is empty → all assertions fail
  * If a pin has no events in VCD → assertion fails
  * Score = assertions passed / total assertions
  * Each assertion is equal weight
